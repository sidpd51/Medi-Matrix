### üöÄ Getting Started

To contribute, follow these steps:

1. **Fork the Repository**  
   Click the **Fork** button in the top right corner of this repository to create your own copy.

2. **Clone Your Fork Locally**
   ```bash
   git clone https://github.com/sidpd51/Medi-Matrix.git
   cd Medi-Matrix
   ```

3. **Create a New Branch**
   ```bash
   git checkout develop
   git checkout -b feature/your-feature-name
   ```

4. **Make Your Changes**  
   Edit the code, fix bugs, improve documentation, or add features.

5. **Run and Test Your Changes Locally**
   ```bash
   cd frontend
   # Install dependencies
   npm install

   # Run the app in development mode
   npm run dev
   ```
   Open your browser at `http://localhost:3000` to see your changes in action.

6. **Commit Your Changes**
   ```bash
   git commit -m "feat: add new feature or fix description"
   ```

7. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

8. **Open a Pull Request**  
   Go to the original repository and open a Pull Request from your fork. Fill out the PR template clearly and concisely.

### ‚úÖ Contribution Guidelines

- Follow the existing code style and conventions.
- Write clear and descriptive commit messages.
- Update or add documentation when needed.
- Test your changes before submitting.
- Link to relevant issues in your PR description when applicable.
